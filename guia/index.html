Factory de vídeos · Node + ffmpeg

Imagem estática vira vídeo cinematográfico

mkVIDEOS transforma imagens geradas por IA em reels e vídeos narrativos com motion grammar via ffmpeg zoompan — sem editor, sem timeline manual, sem IA de vídeo.

Começar o guia Ver no GitHub
O que é

Factory standalone: imagem + motion + áudio → mp4

Um conjunto de scripts Node (CommonJS, sem build) que orquestra geração de imagem, uma "gramática de movimento" de câmera em cima de ffmpeg, e mixagem de áudio para produzir reels 9:16 e vídeos narrativos mais longos, prontos pra publicar.

🎞️ Motion grammar (v2)

16 movimentos de câmera (push-in, whip-pan, flash-cut...) selecionados automaticamente por classe de imagem (rosto, paisagem, dark...) e por papel narrativo (hook, clímax, closing) — sem manipular ffmpeg na mão.

🧩 Gen + Render separados

gen-*.js orquestra ponta a ponta (prompt → imagem → shot plan → render). render-*.js é a receita ffmpeg pura, pra reprocessar um vídeo sem gerar imagens de novo.

🔌 Daemons externos plugáveis

Geração de imagem (inemaimg ou provedores em nuvem), TTS (Chatterbox) e biblioteca de música/SFX (inemavox) rodam fora do repo e são acessados via HTTP, configuráveis por .env.

Como funciona

Do prompt ao mp4 final

O pipeline de um template "gen" típico (ex.: Storytree), da geração das imagens até o vídeo com áudio mixado.

prompts→ gera imagens (inemaimg)→ classifica imagem (face/paisagem/dark...)→ monta shot plan (papel + duração)→ valida (QA)→ zoompan por shot (ffmpeg)→ concat + mux de áudio→ mp4 final

📦 storytree-presets

Vocabulário de 16 movimentos de câmera; exporta as curvas de easing usadas nas expressões zoompan z=.

🎯 storytree-selector

Matriz classe de imagem × papel narrativo → preset, com exclusões (ex.: nada de flash-cut num rosto).

✅ storytree-qa

Valida o shot plan antes de qualquer chamada ao ffmpeg — aborta cedo se algo estiver fora do schema.

Pré-requisitos

O que precisa estar no ar

mkVIDEOS não empacota os daemons de geração de imagem, voz e áudio — eles rodam como infraestrutura separada, acessados por HTTP.

Node 18+ e ffmpeg 6.1+

Única dependência de runtime é dotenv; o resto é stdlib + shell out pro ffmpeg.

# checar versões
node -v
ffmpeg -version

inemaimg (geração de imagem)

API multi-modelo (FLUX.2, Qwen, ERNIE) local, default localhost:8000.

# checar se está no ar
curl -s localhost:8000

Chatterbox TTS + inemavox

TTS local (vozes rachel/bella) em 127.0.0.1:7860 e proxy Freesound + biblioteca de música/SFX em inemavox — só necessários pra templates que mixam narração/áudio.

# .env do projeto
CHATTERBOX_URL=http://127.0.0.1:7860
INEMAVOX_URL=http://192.168.2.99:8010
Guia de uso · passo a passo

Do clone ao primeiro vídeo

Passos reais, na ordem que o README e o CLAUDE.md do repo descrevem.

1

Clone e instale

Dependência única é o dotenv; não há passo de build.

git clone https://github.com/inematds/mkvideos.git && cd mkvideos  # clona
npm install  # instala dotenv
2

Configure o .env

Aponta pros 3 daemons externos e escolhe modelo/voz padrão.

# inemaimg (geração de imagem)
INEMAIMG_URL=http://localhost:8000
INEMAIMG_MODEL=flux2-klein
INEMAIMG_QUALITY=fast

# Chatterbox TTS
CHATTERBOX_URL=http://127.0.0.1:7860
CHATTERBOX_VOICE=rachel

# inemavox (Freesound proxy)
INEMAVOX_URL=http://192.168.2.99:8010
3

Rode o smoke check

Confere sintaxe de todos os scripts antes de gerar qualquer coisa (não há test runner — a validação real acontece em lib/storytree-qa.js no momento do render).

npm run check:syntax  # node --check em todo *.js até depth 3
4

Gere um vídeo ponta a ponta (gen)

Um script gen-*.js orquestra tudo: gera N imagens novas via inemaimg, monta o shot plan e chama o render.

npm run happy  # gen-storytree-happy.js — demo E2E completa
npm run criaprof  # gen-criaprof.js
npm run gertran  # gen-gertran-nostalgia.js
5

Ou só renderize (render), reusando imagens já existentes

Os render-*.js são receitas ffmpeg puras — não pedem imagem nova, só um shot plan + imagens já em output/videos/.... Pra retargetar pra outro conjunto de imagens, edita as constantes SRC/OUT no topo do arquivo (os renderers não são parametrizados por flags, são scripts de uso único).

node render-storytree-v2.js  # motion engine v2 (o mais atual)
node render-darkstory-cinematic-v2.js  # montagem estilo Aronofsky
6

Navegue os outputs pela UI (read-only)

Servidor http vanilla, sem dependências, que só lê — nunca gera nem escreve. Mostra os payloads e previews de imagem/vídeo das campanhas em prj/<projeto>/outputs/<task>.

npm run ui  # localhost:5178
